RakshaNet: Unified AI-Driven Cybersecurity Observability Platform

“Design is not just what it looks like and feels like. Design is how it works.”
— Steve Jobs

Author: Sudip Hazra
Version: 1.0
Date: June 2025

Abstract

RakshaNet is an open, extensible, and AI-powered cybersecurity observability and response platform that integrates log analytics, threat intelligence, and agent-based endpoint monitoring into one unified system. It aims to provide a robust alternative to Splunk and CrowdStrike with modern cloud-native architecture and real-time detection and automation capabilities.

1. Introduction

Security operations face growing complexity, massive data ingestion needs, and increasing attacker sophistication. RakshaNet is designed from first principles to address these using modular AI, clean system design, and full-stack visibility across infrastructure, applications, and cloud APIs.

2. Problem Statement

  • Legacy SIEMs like Splunk are expensive and slow at scale.
  • Most EDRs lack observability or are closed systems.
  • Current solutions separate logging, alerting, and ML — RakshaNet unifies them.

3. Architecture Overview

RakshaNet’s architecture consists of:

  • Raksha Agent: Lightweight binary for log/metric/event collection, EDR-like behavior capture, and endpoint action hooks.
  • RakshaCore (Server): Aggregates all data, runs ML pipelines, stores optimized Delta Lake tables, and exposes APIs/UI.
  • ML Engine: Runs anomaly detection, behavioral profiling, threat scoring, and correlation using open ML models.
  • Cloud Collector: Ingests AWS/GCP/Azure cloud logs, CSPM data, and cloud security posture via APIs.

4. Key Features

  • Streaming & batch ingestion (Apache Spark + Delta Lake)
  • Real-time alerting with rules + anomaly detection
  • Custom detection-as-code support
  • Role-based dashboards & DevSecOps integration
  • EDR-like local agent visibility + control hooks
  • Cloud-native monitoring (IAM, API usage, misconfig detection)

5. Algorithms Used

  • Clustering: K-Means, DBSCAN for anomaly grouping
  • Classification: Random Forest, Logistic Regression
  • Sequential Models: LSTM for user behavior profiling
  • Graph Analytics: PageRank for lateral movement detection
  • Outlier Detection: Isolation Forest, Z-score

6. Agent Design (RakshaAgent)

Inspired by CrowdStrike Falcon’s EDR agent but with open behavior logic and local buffering.

  • Written in Rust (or Go)
  • Captures system metrics, syscall patterns, file/network activity
  • Injects local threat rules and updates via secure channel
  • Can auto-enrich or trigger responses on specific behaviors

7. Cloud Monitoring Features

Native support for:

  • AWS CloudTrail, GuardDuty
  • Azure Activity Logs, Defender for Cloud
  • GCP Admin Activity, VPC Flow Logs
  • IAM misconfiguration detection + excessive privilege alerting
